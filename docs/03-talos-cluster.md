# 03 - Talos Cluster

Single Talos node (combined control-plane + worker), scaling to 3 by adding
entries to `nodes` in `talos/topf.yaml`. Defined declaratively with
[topf](https://postfinance.github.io/topf/): one `topf.yaml` plus layered
patches in `all/`. Talos PKI is committed SOPS-encrypted as the cold DR copy.

The cluster endpoint is a NetBird peer DNS name (`talos-1.netbird.cloud`) and
resolves only when NetBird is up on the client.

## Tools

```bash
brew install siderolabs/tap/talosctl postfinance/tap/topf sops age
```

## Layout

```
topf.yaml          cluster + nodes (connection IPs, versions, schematic)
schematic.yaml     Image Factory schematic (NetBird extension)
secrets.sops.yaml  Talos PKI — generated by topf, SOPS-encrypted, committed
.sops.yaml         age recipients
all/               patches applied to every node
  machine.yaml       machine + cluster config (disk, net, CNI, apiserver)
  certsans.yaml.tpl  adds <host>.netbird.cloud to the Talos API cert SANs
  hostname.yaml.tpl  per-node hostname
  netbird.sops.yaml  NetBird ExtensionServiceConfig + setup key (encrypted)
output/            rendered configs (gitignored — contain secrets)
```

Patches apply in order `all/` → `control-plane/|worker/` → `node/<host>/`.
`.yaml` files get SOPS-decryption; `.yaml.tpl` files get Go templating
(`.Node.Host`, `.Node.Role`). A file is one or the other.

`sops` needs the age key to decrypt. macOS defaults to the wrong path, so set
it (and persist in your shell profile):

```bash
export SOPS_AGE_KEY_FILE="$HOME/.config/sops/age/keys.txt"
```

## One-time setup

```bash
cd talos

# age key (laptop only — back up keys.txt), then paste the age1... public key
# into the `age:` field in .sops.yaml
age-keygen -o ~/.config/sops/age/keys.txt

# NetBird setup key (reusable, from app.netbird.io → Setup Keys). Write the
# ExtensionServiceConfig, then encrypt in place (.sops.yaml matches *.sops.yaml).
cat > all/netbird.sops.yaml <<'EOF'
apiVersion: v1alpha1
kind: ExtensionServiceConfig
name: netbird
environment:
  - NB_SETUP_KEY=<reusable-setup-key>
  - NB_MANAGEMENT_URL=https://api.netbird.io:443
EOF
sops --encrypt --in-place all/netbird.sops.yaml

# Generate the Talos PKI (SOPS-encrypted on disk; stdout dump discarded)
printf 'y\n' | topf secrets > /dev/null

topf render   # validate everything resolves
```

## Provision

1. `dd` the metal image onto the node (see
   [02-dedicated-server.md](02-dedicated-server.md)). It boots into maintenance mode on its public IP, Talos API on :50000.

2. Apply config + bootstrap etcd, then fetch credentials:

   ```bash
   topf apply --auto-bootstrap --submit-to-factory   # --submit-to-factory: first run only
   topf kubeconfig > ~/.kube/homelab.kubeconfig
   topf talosconfig | sed 's/<public-ip>/talos-1.netbird.cloud/g' > ~/.talos/config
   ```

   `topf talosconfig` prints to stdout (it does not write the file) and targets
   the public IP — rewrite it to the NetBird FQDN, which is reachable day-2.

3. Bring NetBird up and talk to the cluster:

   ```bash
   netbird up
   kubectl --kubeconfig ~/.kube/homelab.kubeconfig get nodes
   ```

   Node is `NotReady` until [Cilium](04-cilium.md) is installed.

## Day-2 access

The node's public Talos API (:50000) is **not** reachable from the laptop on a
running cluster, so `topf apply` times out. Render locally and push over NetBird
with talosctl (the cert carries the `talos-1.netbird.cloud` SAN):

```bash
topf render
talosctl apply-config --file output/talos-1.yaml
```

For Talos/K8s upgrades, topf must reach the Talos API: bump versions in
`topf.yaml`, temporarily set the node `ip` to its current NetBird IP
(`dscacheutil -q host -a name talos-1`), run `topf upgrade`, then switch back.

## Cluster endpoint stability

After a node reset NetBird may suffix the new peer (e.g.
`talos-1-25-163.netbird.cloud`) because the old peer still exists. Keep the
endpoint stable by deleting the orphan peer in app.netbird.io → Peers before
re-enrollment, or renaming the new peer to `talos-1`.

## Disaster recovery

The SOPS-encrypted files in git + your age key rebuild the cluster: re-`dd` the
metal image, re-run **Provision**. The PKI in `secrets.sops.yaml` is reused, so
cluster/node identity is preserved.

## Scaling to 3 nodes

Add entries under `nodes` in `topf.yaml` (`host`, `ip`, `role`). Shared settings
live in `all/`; identical hardware needs no per-node patches. `topf apply`
brings up new nodes. Then turn on Cilium `encryption.nodeEncryption`.
